🧭 Athletic Quest – PRD (Product Requirements Document)
🧩 Vision du produit
Athletic Quest (AQ) est une plateforme SaaS gamifiée de bien-être en entreprise.
Elle transforme les bonnes habitudes (activité physique, nutrition, sommeil, respiration, etc.) en une aventure engageante, grâce à un système de points, de niveaux, de challenges et d’objectifs personnalisés.

Notre conviction : le bien-être peut devenir un levier de performance mesurable, tout en redonnant aux collaborateurs le contrôle de leur santé mentale et physique.

🚀 Objectifs principaux
Améliorer la QVT (Qualité de Vie au Travail) de façon mesurable.
Renforcer l'engagement et la cohésion des équipes via des mécanismes ludiques.
Offrir un accompagnement hybride (app + services) aux RH et CSE.
Digitaliser les actions bien-être et les centraliser sur un seul outil.
Utiliser l’IA pour personnaliser les recommandations selon les profils utilisateurs.

🛠 Fonctionnalités clés
🔸 Companion (Web App)
Création de profil joueur avec avatar personnalisable
Suivi des habitudes (sport, nutrition, méditation…)
Système de points d’expérience (XP) et niveaux
Statistiques personnelles et feedback IA
Actions quotidiennes à cocher (+ gains de stats)
Boutique virtuelle AQx (monnaie du jeu)
Historique des séances + retour utilisateur
Intégration IA d'entraînement personnalisé

🔸 Dashboard RH
Gestion des Utilisateurs
Graphique IBEG (Indice de bien-être général) - Courbe de productivité
Bouton Prendre RDV avec Julien 
Suivi de progression globale par équipe

🔸 Dashboard Admin
Gestion des utilisateurs (CRUD)
Attribution de rôles et départements
Suivi de progression globale (par équipe, entreprise)
Création et gestion de clés entreprise
Gestion du système de leaderboard (Intra-entreprise / Inter-départements / Inter-entreprises)

🎯 Cibles
PME / ETI / Grandes entreprises
RH, CSE, Responsables RSE ou QVT

🧱 Stack technique (v1 MVP)
Frontend : HTML/CSS/JavaScript (version no-code possible avec Lovable.dev
Backend : Supabase (auth, base de données, sécurité)
Intégration IA : via JavaScript et localStorage (v1), API personnalisée (v2)
Hébergement : Vercel
Base de données secondaire : Google Sheets API (admin, logs, exports)
